## What's Failing (26/135 modifiers)
The game doesn't tell us WHICH 26 fail, only the total count and the first failure. Likely causes:

1. **Generated hero lines may have subtle format issues** — they were generated by AI agents matching the format of existing lines, but may have small differences the game rejects (wrong property order, missing fields, incorrect tier numbering, etc.)

2. **The textmod format is strict** — the game's parser expects exact syntax. Even one wrong property can cause a modifier to fail.

3. **Approach to debug**: Compare each generated hero line against a KNOWN WORKING line from the original mod, character by character, to find format differences. The original Litten line (Line 37 in the original) is a clean reference.

## Key Format Reference (from working original lines)

A hero line follows this exact structure:
```
hidden&temporary&ph.b[name];1;!mheropool.(TIER1)+(TIER2A)+(TIER2B)+(TIER3A)+(TIER3B).part.1&hidden.mn.[Name]@2!m(skip&hidden&temporary),
```

Each tier is like:
```
replica.[Template].col.[color].tier.[N].hp.[N].[items].sd.[6 faces].img.[sprite]).speech.[cries].n.[Name]
```

Important format details from working lines:
- T1 often has `tier.1` explicitly, T2 often does NOT have `.tier.` (implicit)
- T3 has `.tier.3` explicitly
- The `.part.1&hidden` goes on the LAST T3 variant before `.mn.`
- Property order varies by line but generally: template, col, tier, hp, items, sd, img, speech, n
- The `.mn.` suffix format: `.mn.[Name]@2!m(skip&hidden&temporary),`
- Spell format: `.abilitydata.(Fey.sd.[5 faces].img.[icon].n.[SpellName])`

## How to Debug

1. **Run the validator**: `node tools/validate_textmod.js textmod_heroes_only.txt`
2. **Compare a failing hero against a working original**: Read a generated hero line and the original line it replaced side by side. Look for property order differences, missing fields, wrong tier numbering.
3. **Test incrementally**: Start with original textmod + just ONE hero replacement. If that works, add another. Binary search for the failing lines.
4. **The game is on the user's phone** (mobile app, not desktop). User pastes from VSCode.
